## Remitro Server

신뢰할 수 있는 송금 시스템 **Remitro**는 단순한 금액 이동이 아닌, **정합성과 안정성, 예외 대응 흐름까지 고려한 백엔드 중심의 송금 시스템**입니다.

--- 

### 프로젝트 목적 및 방향

이 프로젝트는 사용자가 간편하게 송금을 주고받을 수 있는 **모의 뱅킹 서비스(Mock Bank)**를 직접 설계하고 구현함으로써,  
**실제 핀테크 서비스에서 요구되는 핵심 백엔드 기술 요소들**을 통합적으로 경험하고자 합니다.

#### 1️⃣ 핀테크 도메인 중심의 시스템 흐름 설계 훈련
- 계좌 생성, 잔고 관리, 송금 처리 등 **실제 금융 서비스에서 사용되는 흐름**을 백엔드 구조로 표현
- 도메인 간 책임 분리, 상태 변화 흐름을 중심으로 구조화

#### 2️⃣ 정합성과 동시성을 보장하는 트랜잭션 처리 학습
- 낙관적 락 / 비관적 락 적용
- 트랜잭션 고립 수준 조절
- 멱등성 처리 (Idempotency Key 활용)

#### 3️⃣ 금융 예외 상황 흐름 설계 및 예외 대응 전략 학습
- 잔고 부족, 자기 자신에게 송금, 존재하지 않는 계좌 송금 등
- 송금 실패 처리, 롤백, 상태 추적 설계 포함

#### 4️⃣ 보안과 인증이 고려된 최소한의 인증 구조 설계
- JWT 기반 인증 처리
- 비밀번호는 `BCrypt`로 암호화
- 주민등록번호 등 민감 정보는 `AES-128` 방식으로 암복호화

---

### 🛠️ 기술 스택

- Java 17
- Spring Boot 3.4.4
- Spring Security / JWT
- JPA / H2
- AES-128 / BCrypt
- Springdoc OpenAPI (Swagger 3.0)
- JUnit5 / Mockito
